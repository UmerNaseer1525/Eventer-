## 🔍 Overview

EventX Backend API is built with:

- **Framework**: Express.js
- **Database**: MongoDB with Mongoose ODM
- **Port**: 3000 (default)
- **CORS**: Enabled for all origins (development mode)

## 📊 Data Models

### User Model

```javascript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  firstName: String,          // Required
  lastName: String,           // Required
  email: String,              // Required, Unique
  password: String,           // Required (store hashed in production)
  role: String,               // Required, enum: ["organizer", "attendee", "admin"]
  phone: String,              // Optional
  profileImage: String,       // Optional (URL)
  status: String,             // Default: "active", enum: ["active", "blocked"]
  createdAt: Date             // Auto-generated
}
```

### Category Model

```javascript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  name: String,               // Required, Unique
  description: String,        // Optional
  createdAt: Date             // Auto-generated
}
```

### Event Model

```javascript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  organizer: ObjectId,        // Required, References User._id
  title: String,              // Required
  description: String,        // Required
  category: ObjectId,         // Required, References Category._id
  location: String,           // Required
  date: Date,                 // Required
  time: String,               // Required
  ticketPrice: Number,        // Required
  capacity: Number,           // Required
  bannerImage: String,        // Optional (URL)
  status: String,             // Default: "draft", enum: ["draft", "published", "cancelled"]
  createdAt: Date             // Auto-generated
}
```

### Booking Model

```javascript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  event: ObjectId,            // Required, References Event._id
  attendee: ObjectId,         // Required, References User._id
  ticketType: String,         // Required, enum: ["VIP", "Regular"]
  quantity: Number,           // Default: 1
  totalPrice: Number,         // Required
  paymentStatus: String,      // Default: "pending", enum: ["paid", "pending", "failed"]
  createdAt: Date             // Auto-generated
}
```

### Payment Model

```javascript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  booking: ObjectId,          // Required, References Booking._id
  transactionId: String,      // Required
  amount: Number,             // Required
  method: String,             // Required, enum: ["card", "jazzcash", "easypaisa"]
  status: String,             // Required, enum: ["success", "failed"]
  paidAt: Date                // Optional
}
```

## ⚠️ Error Handling

### Standard Error Response Format

```json
{
  "message": "Error description here"
}
```

### HTTP Status Codes

- **200**: Success (GET, PUT, DELETE)
- **201**: Created (POST)
- **400**: Bad Request (Invalid input/validation error)
- **404**: Not Found (Resource doesn't exist)
- **500**: Internal Server Error (Server-side error)


#### 500 Internal Server Error

```json
{
  "message": "Database connection failed"
}
```

## 📌 Important Notes

### MongoDB ObjectId

- All `_id` fields are automatically generated by MongoDB
- When creating records, **do not** include `_id` in your request body
- When referencing other documents (foreign keys), use the `_id` value as a string
- Example: `"organizer": "507f1f77bcf86cd799439011"`

### Populated Data

GET endpoints automatically populate referenced documents:

- **Events**: Include organizer and category details
- **Bookings**: Include attendee and event details
- **Payments**: Include booking details with nested attendee and event

### Date Format

- Dates should be sent in ISO 8601 format: `"2026-03-05T10:30:00Z"`
- Or simplified format: `"2026-03-05"`

### Image URLs

- All image fields (`profileImage`, `bannerImage`) expect full URLs
- Example: `"https://example.com/images/banner.jpg"`

### Enum Values

Make sure to use exact values for enum fields:

- **User Role**: `"organizer"`, `"attendee"`, `"admin"` (lowercase)
- **User Status**: `"active"`, `"blocked"`
- **Event Status**: `"draft"`, `"published"`, `"cancelled"`
- **Ticket Type**: `"VIP"`, `"Regular"` (case-sensitive)
- **Payment Status**: `"paid"`, `"pending"`, `"failed"` (Booking)
- **Payment Status**: `"success"`, `"failed"` (Payment)
- **Payment Method**: `"card"`, `"jazzcash"`, `"easypaisa"`

### CORS

- CORS is enabled for all origins in development
- For production, configure specific allowed origins

### Password Security

- **Warning**: Passwords are currently stored as plain text
- **Recommendation**: Implement bcrypt hashing before production
- Add password validation and strength requirements

### Validation

- All required fields are validated on the backend
- Enum values are strictly validated
- Negative values for amounts, prices, quantities, and capacity are rejected

### Partial Updates

- Use specific update endpoints (e.g., `/users/:id/name`) for partial updates
- Use full update endpoints (e.g., `PUT /users/:id`) to update multiple fields at once

### Testing the API

Use tools like:

- **Postman**: Import endpoints from this documentation
- **Thunder Client**: VS Code extension
- **Fetch API**: For React applications

Example Fetch Request:

```javascript
// Create a new user
fetch("http://localhost:3000/api/users", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    firstName: "John",
    lastName: "Doe",
    email: "john@example.com",
    password: "password123",
    role: "attendee",
  }),
})
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((error) => console.error("Error:", error));
```

## 🚀 Getting Started for Frontend Developers

### 1. Start the Backend Server

```bash
cd backend
node app.js
```

Server will run on `http://localhost:3000`

### 2. Test the Connection

Visit `http://localhost:3000/` in your browser to see the API status

### 3. Create API Service in React

Create a service file for API calls:

```javascript
// services/api.js
const API_BASE_URL = "http://localhost:3000/api";

export const api = {
  // Users
  getUsers: () => fetch(`${API_BASE_URL}/users`).then((res) => res.json()),
  getUserById: (id) =>
    fetch(`${API_BASE_URL}/users/${id}`).then((res) => res.json()),
  createUser: (userData) =>
    fetch(`${API_BASE_URL}/users`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(userData),
    }).then((res) => res.json()),

  // Events
  getEvents: () => fetch(`${API_BASE_URL}/events`).then((res) => res.json()),
  getEventById: (id) =>
    fetch(`${API_BASE_URL}/events/${id}`).then((res) => res.json()),
  createEvent: (eventData) =>
    fetch(`${API_BASE_URL}/events`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(eventData),
    }).then((res) => res.json()),

  // Add more API methods as needed...
};
```

### 4. Use in React Components

```javascript
import { useEffect, useState } from "react";
import { api } from "./services/api";

function EventsList() {
  const [events, setEvents] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    api
      .getEvents()
      .then((data) => {
        setEvents(data);
        setLoading(false);
      })
      .catch((error) => {
        console.error("Error fetching events:", error);
        setLoading(false);
      });
  }, []);

  if (loading) return <div>Loading...</div>;

  return (
    <div>
      {events.map((event) => (
        <div key={event._id}>
          <h2>{event.title}</h2>
          <p>{event.description}</p>
        </div>
      ))}
    </div>
  );
}
```
